Cloud Computing (Komputasi Awan) adalah pengiriman layanan komputasi—termasuk server, penyimpanan (*storage*), basis data (*database*), jaringan, perangkat lunak, hingga analitik—melalui internet ("awan"). Tujuan utamanya adalah memberikan inovasi yang lebih cepat, sumber daya yang fleksibel, dan skala ekonomi yang lebih baik.

## Karakteristik Utama

Secara fundamental, sebuah layanan dapat disebut sebagai Cloud Computing jika memenuhi lima karakteristik penting dari NIST:

1. **On-demand Self-service:** Pengguna dapat menyediakan komputasi secara sepihak dan otomatis kapan pun dibutuhkan tanpa harus berinteraksi dengan staf IT dari penyedia layanan.
2. **Broad Network Access:** Kapabilitas tersedia di jaringan dan diakses melalui mekanisme standar yang dapat digunakan oleh berbagai jenis perangkat (seperti PC, tablet, atau *smartphone*).
3. **Resource Pooling:** Sumber daya komputasi penyedia layanan digabungkan untuk melayani banyak pengguna (model *multi-tenant*), dengan sumber daya fisik dan virtual yang dialokasikan secara dinamis sesuai permintaan.
4. **Rapid Elasticity:** Kemampuan dapat dengan cepat disesuaikan secara elastis, baik ditambah (*scale out/up*) maupun dikurangi (*scale in/down*) sesuai dengan beban kerja.
5. **Measured Service:** Sistem secara otomatis mengontrol dan mengoptimalkan penggunaan sumber daya, di mana pengawasan (metering) digunakan untuk sistem pembayaran terukur (*pay-as-you-go*).

## Knowledge Base (KB) yang Harus Dikuasai

Bagi seorang siswa atau calon praktisi cloud, mempelajari cloud tidak hanya soal menekan tombol di *dashboard*. Ada beberapa pilar pengetahuan (Knowledge Base) pendukung yang sangat penting dikuasai:

- **Jaringan Komputer (Networking):** Memahami subnetting, IP Address (Public vs Private), Routing, DNS, Firewall, dan NAT. (Tanpa dasar jaringan, kamu akan kesulitan mengatur VPC).
- **Sistem Operasi (Linux):** Mayoritas server di cloud menggunakan Linux. Kemampuan menggunakan Command Line Interface (CLI), manajemen hak akses (*permissions*), dan struktur direktori sangat krusial.
- **Keamanan (Security):** Prinsip perlindungan data, manajemen identitas (IAM), enkripsi, dan *security groups*.
- **Virtualisasi & Container:** Memahami bagaimana mesin virtual dan container (seperti Docker) bekerja di latar belakang.
- **Basis Data (Database):** Pemahaman dasar antara Relational Database (seperti MySQL/PostgreSQL) dan NoSQL.

Menguasai dasar-dasar di atas akan membuat proses adaptasi dalam menggunakan layanan Cloud (seperti AWS, Azure, atau Google Cloud) menjadi jauh lebih mudah dan masuk akal.
